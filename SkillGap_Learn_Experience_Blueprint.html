The Learn Experience

End-to-end blueprint for SkillGap.ai's guided learning product — March 2026

The Core Belief

What We Believe
People don't need more content. They need guided, hands-on practice with the tools that will make them indispensable.
YouTube has infinite AI content. ChatGPT can answer any question. SkillGap's value is not finding videos or answering questions — it's designing a curriculum that takes someone from "I've heard of this" to "I can do this" through structured, task-driven practice with real AI tools. Content exists in service of tasks. Tasks are the product.

The Two Entry Modes

Every user enters through one of two doors. Both lead to the same guided experience.

Mode 1 — Pull (Intent)

"I want to learn Cursor"

User knows what they want. They search or browse by topic. They're looking for a structured path to get hands-on fast. Entry: search bar, topic filter, or direct link.

What they see: Learning paths matching their search. Each path has a clear promise ("Build AI-powered apps with Cursor"), time estimate, difficulty level, and a "Start" button.

Mode 2 — Push (Discovery)

"What should I know about right now?"

User wants to stay sharp but doesn't have a specific tool in mind. They need SkillGap to surface what matters. Entry: Learn homepage, digest notification, or personalized recommendation.

What they see: Curated discovery feed. Trending topics. Personalized recommendations. Emerging tools. Each card = a learning path, not a video. Every card leads to the same guided experience.

The critical rule: No matter how a user arrives — search, discovery, recommendation, shared link — they always land on a learning path with a clear goal, structured tasks, and integrated tool practice. Never a video. Never a playlist. Never a content library.

The Curriculum Model

This is the fundamental architecture. Everything else follows from this.

The Shift
Old model: Find videos → sequence them → bolt on practice.
New model: Define goal → design tasks → attach supporting content.
The primary unit of the curriculum is a task — something the user DOES. Videos, articles, and tutorials are supporting material attached to tasks. The AI curriculum generator designs a task sequence that builds progressively toward the goal, then finds the best content from the lake to support each task.

Anatomy of a Learning Path

LayerDescription
Goal What the user will be able to DO when they finish. Not "understand X" but "build X with Y tool." Shown prominently on the path card and entry screen. Example: "Build and deploy an AI-powered app using Cursor."
Modules 3-6 progressive stages. Each module has a theme and builds on the previous. Module 1 might be orientation + setup. Module 3 might be guided building. Module 5 might be a solo challenge. Each module = 10-20 minutes.
Tasks The atomic unit. Each task is one of four types: Watch (consume supporting content), Do (hands-on with a tool), Check (verify understanding), or Reflect (connect to personal context). Tasks within a module are sequenced intentionally — typically Watch → Do → Check, with Reflect at module boundaries.
Supporting Content Videos and articles from the content lake, attached to Watch tasks. These are not the point — they support the Do tasks. A video might show someone doing the exact thing the user is about to do, so they have a reference. Shown in a reference panel, not full-screen.
Tool Integration Each Do task names the tool, provides a pre-built prompt (copyable), deep-links where supported, and defines success criteria. The tool is part of the task, not a side quest.

The Four Task Types

👁 Watch

Consume Supporting Content

Purpose: Give context before a Do task. Show what "good" looks like. Introduce a concept needed for the next step.

UI: Video or article in a content viewer. Key moments highlighted. Duration: 3-8 min max. Always followed by a Do or Check task — never two Watch tasks in a row.

Rule: Every Watch task must answer: "Why am I watching this?" The answer should be: "Because in 3 minutes, you're going to do this yourself."

🔨 Do (Tool Mission)

Hands-On with a Real Tool

Purpose: The main event. User opens an AI tool and completes a specific task. This is what makes SkillGap different from YouTube.

UI: Mission briefing (objective, context, pre-built prompt, steps, success criteria, tool CTA). Clipboard copy on launch. Persistent reference bar while they work. Submit output (paste or screenshot). AI evaluation with specific feedback.

Rule: Proof of work required for credit. Skip available but tracked. Iteration encouraged ("Try again" on non-perfect scores).

✓ Check

Verify Understanding

Purpose: Quick validation that concepts stuck before moving on. Not a quiz — a checkpoint.

UI: Inline overlay at the bottom of the content viewer (not a separate step). 1-2 questions. Multiple choice or short free-text. Wrong = hint + retry (max 2 attempts, then reveal). Right = micro-animation + auto-advance.

Rule: Not skippable, but never blocks for more than 30 seconds. Lightweight. The point is confirmation, not assessment.

💭 Reflect

Connect to Personal Context

Purpose: Bridge between learning and the user's real work/goals. "How would you apply this?" Appears at module boundaries.

UI: Clean textarea with personalized prompt (references their role/goal from profile). Min 50 chars. AI responds conversationally, not as a grader. Saved as a learning note. Always advances.

Rule: No grade. No judgment. This is about making the learning personal and sticky. Optional save to profile.

A Complete User Journey

One path, end to end: "Build AI-Powered Apps with Cursor." Every detail.

Context: User is a junior developer or PM who's heard of Cursor but hasn't used it. They searched "Cursor" on SkillGap, saw this path, and clicked "Start." Goal: go from zero to shipping something real.

Module 1: See What's Possible

~8 min
👁
Watch: "What Cursor Can Do" (4 min)
Curated video showing real-world Cursor usage. Someone building an app in minutes. This isn't education yet — it's motivation. The user sees the destination.
✓
Check: Quick comprehension (inline)
"What makes Cursor different from a regular code editor?" Multiple choice. Validates they absorbed the core concept before moving on.
🔨
Do: Install and open Cursor
First hands-on task — happens within 5 minutes of starting. "Download Cursor from cursor.com, open it, and take a screenshot of the welcome screen." Deep link to cursor.com. Submit screenshot. AI vision confirms. Quick win.

Module 2: Your First AI-Generated Component

~15 min
👁
Watch: "How Cursor's Composer Works" (3 min)
Short, targeted video showing exactly the feature they're about to use. Shown in a reference panel — not fullscreen. They can watch it while reading the mission briefing.
🔨
Do: Generate a login form with Cursor Composer
Mission briefing:
Objective: "Use Cursor's Composer to generate a complete React login form from a single prompt."
Pre-built prompt (shown in copyable code block): "Create a React login form component with email and password fields, form validation, error states, and a submit button. Use Tailwind CSS for styling."
Steps: 1) Open Cursor. 2) Create a new file. 3) Open Composer (Cmd+I). 4) Paste the prompt. 5) Review and accept the output.
Success criteria: ✓ Has email + password fields ✓ Has validation ✓ Has error states ✓ Uses Tailwind
CTA: "Open Cursor" (copies prompt to clipboard + toast)

Submission: Paste generated code. AI evaluates against criteria. Specific feedback: "Your form has all required fields ✓ but is missing client-side validation ✗. Try adding this to your prompt: 'Include email format validation and password strength indicator.'"
✓
Check: What did you learn? (inline)
"What's the most effective way to get better output from Cursor?" Free-text, AI-evaluated. Validates they're thinking about prompt quality, not just copy-pasting.

Module 3: Edit and Iterate

~15 min
👁
Watch: "Cursor's Inline Edit + Multi-File Edits" (4 min)
Shows Cmd+K inline editing and multi-file generation. Reference for the task ahead.
🔨
Do: Improve your login form with 3 iterations
Builds directly on Module 2's output. "Take your login form and make three improvements using Cursor's inline edit (Cmd+K)."
Iteration 1: "Add a 'Remember me' checkbox and 'Forgot password' link"
Iteration 2: "Add a loading spinner on the submit button"
Iteration 3: "Add social login buttons (Google, GitHub)"
Each iteration has its own prompt provided. Submit final code. AI evaluates all three additions.
💭
Reflect: Module milestone
"You've now generated AND iterated on a component using AI. How could you use this workflow in your current work?" Personalized to their role from profile. Saved as learning note.

Module 4: Full-Stack with AI

~20 min
👁
Watch: "Multi-File Generation in Cursor" (5 min)
Shows someone building a frontend + API + database schema in one Composer session. This is the reference for the biggest task in the path.
🔨
Do: Build a complete auth flow
The big one. "Use Cursor to generate: 1) Your login form (you already have this), 2) A registration form, 3) An API route that handles login/register, 4) A protected dashboard page."
One comprehensive prompt provided. But they also need to iterate — the first output won't connect everything correctly. This is where they learn the real skill: multi-turn iteration with AI.
Submit: All 4 files. AI evaluates the connections between them.

Module 5: Solo Challenge

~20 min
🔨
Do: Build a user profile page — no prompts provided
This is the test. "Build a user profile page that fetches data from an API and displays it. Use Cursor. Write your own prompts. Make it look good."
No pre-built prompt. No step-by-step. They compose their own prompts, iterate, and produce working code. This proves they've internalized the skill.
Submit: Final code. Detailed AI evaluation covering: prompt quality (inferred from output), code completeness, UI polish, API integration.
💭
Reflect: Path completion
"What was the biggest thing you learned about working with AI coding tools? How will you use this going forward?"
🏆
Achievement: "Cursor Builder — Level 1"
Completion card with what they built, skills demonstrated, time spent. Shareable to LinkedIn/X. Public achievement URL. Added to skill profile.

How Discovery Powers the Feed

The pipeline from "something is trending" to "there's a path about it on the homepage."

1
Signals Detected
YouTube trending, Reddit discussion, HN post, new tool launch, RSS from AI blogs
2
Topic Clustered
Signal system groups signals into topic clusters. Heat score calculated. "Cursor background agents" = hot topic.
3
Content Sourced
Content lake searched for quality videos/articles on the topic. If thin, discovery cron fetches more from YouTube.
4
Curriculum Generated
AI designs a task sequence for the topic. Attaches best content from lake as support. Generates prompts, success criteria, tool integrations.
5
Editorial Review
Queued for human QA. Approved paths go live. Rejected paths get notes for improvement. (Auto-publish later as confidence grows.)
6
Live on Feed
Path appears in "What's Moving Right Now" with topic context, time estimate, and "Start" button.

Two Path Speeds

Pre-Generated Paths

Discovery Feed + Popular Topics

Generated daily by cron (or triggered by hot signal). Full editorial review. These are the polished, high-confidence paths that appear on the homepage. Cover trending topics and evergreen skills.

On-Demand Paths

Search Results + Long-Tail Topics

Generated in real-time when a user searches for something without a pre-made path. Uses content lake + curriculum generator. Quality may vary — flagged for async editorial review. User can start immediately.

The Curriculum Generator

The AI brain that designs task sequences. This is the biggest architectural change from the current system.

Current Path Generator (What Exists)

Content-First Model

Find → Sequence → Decorate

pathGenerator.ts searches the content lake for relevant videos → sequences them by topic progression → adds connective text between items → bolts on practice from scaffolding data. The curriculum IS the video sequence. Practice is afterthought decoration.

New Curriculum Generator (What We Need)

Goal-First Model

Goal → Tasks → Content

New curriculum generator takes a GOAL (topic + skill level + desired outcome) → designs a MODULE sequence with progressive tasks → for each Watch task, finds the best supporting content from the lake → for each Do task, generates the mission briefing (objective, prompt, steps, success criteria, tool integration) → for each Check task, generates contextual questions. The curriculum IS the task sequence. Content is attached support.

InputOutput

Available content: 12 quality videos from lake
Available tools: Cursor (from tool registry) 		Path goal: "Build and deploy an AI-powered app using Cursor"
Modules: 5 modules, each with 2-4 tasks

Key Design Principle: Progressive Scaffolding Removal

Early
Fully Guided
Complete prompt provided. Step-by-step instructions. Reference video available. Success criteria explicit.
Middle
Scaffolded
Partial prompt provided. High-level steps only. Video optional. Success criteria still explicit.
Late
Independent
No prompt provided. Goal only. User composes their own approach. AI evaluates the output, not the process.
Final
Solo Challenge
Open-ended. "Build X." No guidance. Proves mastery. Achievement unlocked.
