3. Identified Token Gaps

Gap 1: No Error / Danger / Red Token Resolved

The new --sg-* system has forest (green), teal (blue-green), gold (warm), sage (muted green), and shell (neutral). There is no red anywhere in the palette. The old system used two reds:

Old TokenHexWhere Used
CORAL_700 / --color-coral-700 #E96861 Advanced difficulty badge, danger button variant, error accent, Ember character primary, "end session" button, delete confirmations
RED_700 / --color-red-700 #EF5555 Priority-urgent marker

Both reference mockups had to fall back to raw #E96861 — the only hardcoded hex in either file.

Recommendation: Add a minimal red/coral scale to the token system. It doesn't need 10 steps — 4 is enough for most UI needs:
Proposed TokenHexUsage
--sg-red-600#D64540Error text, danger button bg
--sg-red-500#E96861Default error/danger color (maps from coral-700)
--sg-red-100#FDECEBError tint background
--sg-red-on-darkrgba(233,104,97,0.2)Error tint on forest-dark surfaces

Alternatively, name them --sg-coral-* to feel less clinical and more on-brand. Either way, this is a functional requirement — error states and destructive actions need a red.

Gap 2: Character Territory Theming Medium

The current app has three AI characters (Ember, Moss, Byte), each with a 4-token color territory:

CharacterPrimarySecondaryAccentTint BG
Ember #E96861 (coral) #EE984B (orange) #F5C54E (gold) #FEF8F7
Moss #5BC682 (green) #5CC2EC (cyan) #ADE2C0 #F7FCF9
Byte #535AEE (purple) #8C55EF (violet) #4636D3 #F6F7FE

In the new system, Moss maps naturally to forest/teal. Ember partially maps to gold (but loses its red primary). Byte has no home — purple/violet are completely gone.

Recommendation — choose one of three strategies:
  1. Retire character colors. All AI hosts use a single neutral treatment (forest-tinted glass), differentiated by avatar/name only. Simplest option. Reduces the token surface by 12 values.
  2. Re-map to the new palette. Ember → Gold (primary: gold-600, secondary: gold-500). Moss → Forest (primary: forest-400, secondary: teal-500). Byte → Teal (primary: teal-600, secondary: teal-400). Narrower differentiation but stays within the system.
  3. Keep character colors as override tokens. Define --sg-char-ember-*, --sg-char-moss-*, --sg-char-byte-* as a separate layer that overrides the base system. Most flexible, most complex.

Strategy 1 (retire) is recommended given the brand shift away from playful multi-color toward professional forest-first. Characters can still feel distinct through tone, avatar, and naming without needing unique color territories.

Gap 4: Fluency Badge On-Dark Variants Low — Solved in Ref

Fluency levels (Exploring, Practicing, Proficient, Advanced) each have a badge color. On light backgrounds, the brand kit tokens work directly. On dark (forest-900/800) backgrounds, the ref mockups used rgba tints:

LevelOn-LightOn-Dark (rgba tint)

Recommendation: Document the on-dark recipe in the brand kit as a pattern, not 8 additional tokens. The recipe is: background: rgba([base-color], 0.15); color: [lighter-step]. This keeps the token count manageable while giving implementers a clear rule.

Gap 5: Progress Bar Gradient Recipe Low

Learning path progress bars in the current app use a simple fill. The ref mockups used a forest-400 → teal-500 gradient for visual interest. This works well but isn't formally defined.

Recommendation: Add a "progress gradient" recipe to the brand kit: background: linear-gradient(90deg, var(--sg-forest-400), var(--sg-teal-500)). Used for: progress bars, completion indicators, skill graph fill. Optional — solid forest-400 is the simpler fallback.

Gap 7: Orange Scale Consolidation Low

The old system had separate orange (#EE984B) and amber (#F59E0B) scales. The new system has only gold. Orange is currently used for PRIORITY_HIGH and accent-warning.

Recommendation: Map both to --sg-gold-600 / --sg-gold-500. If a visual distinction between "warning" and "high priority" is needed, use gold-600 for warning and gold-900 (dark text on gold-100 bg) for priority. This is a minor semantic collapse that shouldn't cause confusion.
