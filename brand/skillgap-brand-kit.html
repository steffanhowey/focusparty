Application Typography Rules
Headings (Inter 600–700)
All headings use Inter. Weight 700 for page titles, 600 for section headings and card titles. Letter-spacing −0.025em at large sizes, −0.01em at medium. Size creates hierarchy — not font family.
Body (Inter 400–500)
All body, labels, navigation, buttons. Weight 400 for body text, 500 for interactive elements (buttons, links, nav). Line-height 1.5 minimum. Optimized for 8-hour reading at 12–16px.
Why Inter?
Built by Rasmus Andersson specifically for screens. Tall x-height, open apertures, optical sizing axis. Clean hinting at 11–14px where product UI lives. Used by Linear, Figma, Notion.
Why not Fraunces in-app?
Display fonts optimize for impression (you see it once). Product fonts optimize for duration (you live in them). Distinctive letterforms compound cognitive load over 8-hour sessions. User feedback confirmed this.

Nesting Rule

When an element sits inside a rounded parent with padding between them, the inner radius must account for the padding: inner = max(0, outer − padding). If the math yields ≤ 0, use radius-xs (4px) as the minimum.

padding: 16px

Card → Input inside
outer: 12px (md)
padding: 12px
inner: max(0, 12−12) = 4px (xs)
Section → Card inside
outer: 24px (xl)

Shell Principle Rules

Shell Principle

  • Topbar, sidebar, nav — always shell neutrals
  • Canvas/page background: Shell 200 (#EEEEEE)
  • Panels and cards: Shell White (#FFF) on Shell 200
  • Borders: Shell Border (#E8E8E8) — one weight only
  • Shell text: Shell 900 for primary, Shell 600 for secondary
  • Brand color ONLY on interactive elements within the shell

Avoid

  • Forest green in the topbar or sidebar background
  • Gold or teal in navigation chrome
  • Multiple border weights or colors in the shell
  • Brand-colored backgrounds for structural layout
  • Shell colors for achievement badges or status indicators
  • Shadows heavier than 4px blur in the shell

Email Rules & Suppression Logic
Always
System font stack — no web fonts, ever
600px max width, single column
48px minimum CTA button height
One primary CTA per email
Off-white outer (#F5F5F5), white content
One-click unsubscribe on all non-auth emails
Max 1 email per day per user
Never
Custom web fonts (fail silently in most clients)
Flexbox or CSS grid (use tables)
Pure white (#FFF) or pure black (#000) — breaks dark mode
More than 2 CTAs in a single email
Images as primary content (use as enhancement only)
JavaScript (all clients strip it)
Subject lines over 50 characters
Suppression Logic
Welcome: Immediate, no suppression
First Path Nudge: Suppress if path already started
Weekly Digest: Suppress if active in last 24h
Lapsed: Once after 14 days, don't repeat for 30 days
Milestone: Max 1 per week even if multiple milestones hit
Surge Alert: Max 2 per month, only for user's function
Credential Share: Sent 1h after completion (give in-app moment space)
